## 核心設計特色（v3.1）

| 特色 | 說明 | 效益 |
|------|------|------|
| **雙向運作** | 收件（外送→住戶）+ 寄件（住戶→快遞） | 一套系統雙向服務 |
| **箱體即容器** | 包裹放入箱體後不再移動，箱體整體搬運 | 避免卡住、摔落、混淆 |
| **正開箱體** | S/M 兩種尺寸，寬面開口放取方便 | 深度僅 60cm，適合走廊 |
| **單升降機雙列** | 一個升降機服務左右兩列暫存位 | 減少成本、減少故障點 |
| **統一開口** | 投件口/取貨口共用設計 | 模組共用，開發成本降低 |
| **AMR 封閉艙體** | 包裹運送全程封閉保護 | 隱私 + 防盜 + 防塵 |

## 開發時程

### Phase 1：MVP 驗證

| 週數 | 工作項目 | 負責 |
|------|----------|------|
| 1-4 | 選定 AMR 廠商（封閉艙體 + 側門對接） | PM + 機構 |
| 1-4 | 雲平台基礎架構 | 雲平台 |
| 5-8 | 升降機模組 PoC | 機構 + 電控 |
| 5-8 | 嵌入式韌體開發 | 軟體 |
| 5-8 | AMR 調度 API 開發 | 雲平台 |
| 9-12 | 側門對接測試 | 機構 + 電控 + 軟體 |
| 9-12 | 端到端整合測試 | 全體 |

### Phase 2：場域驗證

- 示範場域實際運作
- 數據收集與優化
- 使用者回饋收集（PM）

### Phase 3：產品化

- 產品定型、認證申請
- 量產準備（機構 + 電控）
- 正式版雲平台上線
