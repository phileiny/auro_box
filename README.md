# 智慧社區包裹配送系統 (Auro Box)

**SWAT Team 專案** — 智慧建築包裹配送解決方案

採用 AMR 封閉艙體 + 統一投件/取貨口設計，目標市場為台灣智慧建築容積獎勵認證。

## 核心設計特色

| 特色 | 說明 | 效益 |
|------|------|------|
| **統一投件口** | 外送員只需放入一個固定位置 | 10 秒完成投件，不用找格子 |
| **統一取貨口** | 住戶只需在一個固定位置取件 | 5 秒完成取件，不會開錯 |
| **AMR 封閉艙體** | 包裹運送全程封閉保護 | 隱私 + 防盜 + 防塵 |
| **模組化設計** | 升降機、櫃格、投件口大廳站/樓層站共用 | 開發成本降低 30-40% |
| **一趟一件** | AMR 每趟只載一件包裹 | 絕對不混淆，軟硬體簡單 |

## 團隊分工

### 角色與負責範圍

| 角色 | 負責範圍 |
|------|----------|
| **產品經理 (PM)** | 需求定義、市場驗證、產品規格、客戶溝通 |
| **專案經理 (PjM)** | 時程管理、資源協調、風險控管、進度追蹤 |
| **機構** | 櫃體結構、升降機模組、投件/取貨口、底座框架、AMR 艙體規格 |
| **電控** | 馬達驅動、感測器整合、RS485 通訊、主控/子控板設計 |

| **雲平台** | 後端 API、AMR 調度、住戶通知、數據報表、電梯整合 |

## 開發時程

### Phase 1：MVP 驗證

| 週數 | 工作項目 | 負責 |
|------|----------|------|
| 1-4 | 選定 AMR 廠商（封閉艙體 + 側門對接） | PM + 機構 |
| 1-4 | 雲平台基礎架構 | 雲平台 |
| 5-8 | 升降機模組 PoC | 機構 + 電控 |
| 5-8 | 嵌入式韌體開發 | 軟體 |
| 5-8 | AMR 調度 API 開發 | 雲平台 |
| 9-12 | 側門對接測試 | 機構 + 電控 + 軟體 |
| 9-12 | 端到端整合測試 | 全體 |

### Phase 2：場域驗證

- 示範場域實際運作
- 數據收集與優化
- 使用者回饋收集（PM）

### Phase 3：產品化

- 產品定型、認證申請
- 量產準備（機構 + 電控）
- 正式版雲平台上線
