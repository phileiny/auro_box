  |------------------|-----------------------------------|
  | 2.4G 原始封包        | 不會直接傳到雲平台                         |
  | air_says 消息內容    | ✅ 會傳到雲平台 (Base64 編碼)              |

  | 上報機制             | Robot 透過 4G SyncMeteorEvents 打包上報 |
  | 上報頻率             | 每 1-5 秒 (隨心跳一起)                   |

  關鍵點：
  1. 2.4G 通訊發生在 Station ↔ Robot 之間（本地）
  2. Robot 收集這些 2.4G 消息，打包成 air_says 陣列
  3. 透過 4G 的心跳/事件同步機制上報給雲平台
  4. 雲平台的 Actor 可以看到這些消息的內容（Base64 編碼）
