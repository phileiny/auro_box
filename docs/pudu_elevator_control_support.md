## 梯控整合方案

普渡提供兩種梯控方案：

### 1. 雲端梯控方案 (Cloud Elevator Control)

- 無需電梯硬體改造，部署簡便
- 透過 API 與電梯系統雲端對接
- 已與 Otis（奧的斯）完成整合測試，使用 Otis Integrated Dispatch 技術
- 兼容品牌：KONE、OTIS 等主流電梯

### 2. 硬體梯控方案 (Hardware Elevator Control)

- 普渡自研方案
- 採用輕量級改造方式
- 不控制、損壞或參與電梯內部系統
- 模擬人類行為完成乘梯過程（按鍵、識別樓層）
- 獨立且安全的電梯物聯網方案

## 梯控相關 API（T300 雲端平台）

### 回調通知

| API | 說明 |
|-----|------|
| `notifyElevatorUtilizeState` | 機器梯控通知，監控乘梯狀態 |
| `notifySwitchMap` | 切換地圖的任務通知 |

### 控制指令

| API | 說明 |
|-----|------|
| 電梯內切換當前使用的樓層地圖 | 多樓層場景下的地圖切換 |
| 切換地圖 | 手動切換機器人使用的地圖 |

---

## 備註

- FlashBot 是普渡主打的樓宇配送機型，梯控功能最完整
- T300/T600 工業機型的梯控主要針對工廠/倉庫的貨梯場景
- 餐飲類機器人（BellaBot、KettyBot 等）設計上以單層場景為主
- 實際部署梯控功能需與普渡或和椿確認當地電梯品牌兼容性
