## 1. 專案概述

### 1.1 市場背景

**目標市場**: 台灣智慧建築容積獎勵認證

根據 2024 年智慧建築評估手冊：
- 容積獎勵：合格 2%、銅級 4%、銀級 6%、金級 8%、鑽石級 10%

### 1.2 設計理念

**核心原則**: 
1. 移除「智取櫃機械手」- 成本高、故障率高
2. 採用「一趟一件」- 避免包裹混淆
3. **全系統模組化** - 櫃格模組、升降機模組大廳站/樓層站共用
4. **垂直升降取代斜坡滑道** - 體積更小、不會卡住、速度可控
5. **統一投件口** - 外送員不需找格子，10 秒完成投件
6. **統一取貨口** - 住戶不需找格子，5 秒完成取件
7. **AMR 封閉艙體** - 包裹運送全程保護，隱私+防盜
8. **食品格 = 一般 M 格** - 無特殊設計，軟體層面優先調度

### 1.3 設計特色

| 特色 | 說明 |
|------|------|
| **統一投件口** | 外送員只需放入一個固定位置，系統自動分配 |
| **統一取貨口** | 住戶只需在一個固定位置取件，系統自動送出 |
| **AMR 封閉艙體** | 包裹運送全程封閉保護，隱私+防盜+防塵 |
| **全系統模組化** | 櫃格、升降機、底座模組可共用 |
| **升降機共用** | 大廳站/樓層站使用同款升降機模組 |
| **無斜坡設計** | 垂直升降，體積小，不卡住 |
| **無特殊食品格** | 食品走同一路徑，全程水平不摔落 |
| **一趟一件** | AMR 每趟只載一件，絕對不混淆 |

### 1.4 使用者體驗摘要

| 角色 | 操作 | 時間 |
|------|------|------|
| **外送員** | 掃碼 → 放入統一投件口 → 完成 | 10 秒 |
| **住戶** | 掃碼/一鍵取件 → 從統一取貨口取走 → 完成 | 5 秒 |
| **包裹** | 全程在封閉艙體內運送 | 安全 |

### 4.1 為什麼用升降機取代斜坡滑道？

```

    ✅ 不需要緩衝墊
    ✅ 全程水平，食品不會灑

### 6.1 設計原則

```
樓層站採用「對稱設計」：

• 收貨口（後方）：接收 AMR 送來的包裹

## 13. 設計決策總結

| 決策項目 | 選擇 | 原因 |
|----------|------|------|
| 機械手 | ❌ 不用 | 成本高、故障率高 |
| 批次配送 | ❌ 不用 | 會混淆包裹 |
| 一趟一件 | ✅ 採用 | 簡單可靠 |
| 斜坡滑道 | ❌ 不用 | 體積大、會卡住、會摔落 |
| **垂直升降機** | ✅ 採用 | 體積小、速度可控、不摔落 |
| **升降機共用** | ✅ 採用 | 大廳站/樓層站用同款模組 |
| 外送員找格子 | ❌ 不用 | 浪費時間、可能放錯 |
| **統一投件口** | ✅ 採用 | 10 秒完成、不會放錯 |
| 住戶找格子 | ❌ 不用 | 浪費時間、可能開錯 |
| **統一取貨口** | ✅ 採用 | 5 秒完成、不會開錯 |
| AMR 開放式 | ❌ 不用 | 包裹裸露、不安全 |
| **AMR 封閉艙體** | ✅ 採用 | 隱私+防盜+防塵 |
| 特殊食品格 | ❌ 不用 | 升降機全程水平，不需特殊設計 |
| 食品優先（軟體）| ✅ 採用 | M 格 + 軟體標示即可 |
| **側門對接** | ✅ 採用 | AMR 封閉艙體的對接方式 |
| AMR 數量 | 2 台 | 冗餘備援 |
